# Databases and Analytics Coursework – NorthStar Urban Mobility and Logistics

This repository contains the coursework submission for the Databases and Analytics module. The project is based on the NorthStar Urban Mobility and Logistics case study and focuses on identifying the causes of poor operational performance, customer complaints, delays, service failures, and data fragmentation across the organisation. 

## Project Overview

The aim of this coursework is to apply database and analytics techniques to a realistic business problem using multiple tools and technologies. The analysis combines relational querying, statistical analysis, Python-based data processing, and MongoDB document design to investigate operational inefficiencies and recommend improvements to NorthStar’s data architecture.

## Repository Contents

This repository includes the following work:

- SQL within R analysis
- R analytics and visualisation
- Python data processing and visualisation
- MongoDB Atlas design and querying
- Query optimisation and indexing
- Supporting screenshots and outputs where required [file:13]

## Notebook Structure

The coursework is organised into the following sections:

### 1. SQL within R
This section demonstrates how SQL queries were used inside R to analyse structured NorthStar operational data. It includes filtering, aggregation, grouping, and SQL data manipulation operations such as INSERT, UPDATE, and DELETE. [file:13]

### 2. R Analytics and Visualisation
This section uses R to explore delivery performance, complaints, incidents, customer ratings, route overrides, and delivery cost patterns through summary analysis and visualisation. All figures are interpreted in relation to the NorthStar business case. 

### 3. Python Data Processing and Visualisation
This section uses Python libraries such as Pandas, NumPy, Matplotlib, and Seaborn to clean, merge, analyse, and visualise the NorthStar datasets. It includes data quality checks, feature creation, descriptive statistics, grouped analysis, and pairwise numerical plots. 

### 4. MongoDB Atlas Design and Implementation
This section presents the MongoDB solution for the more flexible and event-based parts of the NorthStar case study. It includes document design, data insertion, CRUD operations, nested querying, and justification for using a NoSQL database. 

### 5. Query Optimisation
This section demonstrates indexing and query optimisation techniques in MongoDB. It includes index creation, discussion of performance improvement, and explanation of how optimisation supports faster retrieval of operational records.
